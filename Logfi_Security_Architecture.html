1. סקירה כללית של המערכת

Logfi היא מערכת ניהול פיננסי פנימית שנבנתה עבור לוגי גרופ / מנרב IFM. המערכת מנהלת את הצד הכספי של פרויקטי בנייה, תחזוקה וניהול מתקנים — כולל דוחות רווח והפסד, תזרים מזומנים, ניהול קבלני משנה, תנאי תשלום וניתוח סיכונים.

סוגי נתונים במערכת

  • נתוני פרויקטים: שם, תאריכים, הכנסות, הוצאות, תנאי תשלום
  • נתוני ספקים: שם, פרטי קשר, סכומי התחייבות, תנאי תשלום
  • עלויות שכר: סכומים חודשיים לפרויקט (לא שכר אישי של עובדים)
  • דוחות פיננסיים: רווח והפסד, תזרים מזומנים, ניתוח סיכונים
  • נתוני משתמשים: שם, אימייל, תפקיד, הרשאות

3. אבטחת מידע — אימות והרשאות

אימות (Authentication)

  • אימות מבוסס Firebase Authentication
  • התחברות באמצעות אימייל + סיסמה
  • אימות דו-שלבי (2FA) באמצעות Google Authenticator — פרוטוקול TOTP
  • טוקן JWT נוצר על ידי Firebase, מאומת בצד השרת עם Firebase Admin SDK
  • תוקף טוקן: 60 דקות — מתחדש אוטומטית
  • סיסמאות מאוחסנות ב-Firebase בלבד — לא בקוד ולא בבסיס נתונים של המערכת

הרשאות (Authorization)

  • 4 רמות הרשאה: מנהל מערכת, כלכלנית, צופה, מנהל פרויקט
  • כל בקשת API מאומתת — טוקן נבדק בכל endpoint
  • מנהל פרויקט רואה אך ורק פרויקטים שמשויכים אליו
  • בדיקת הרשאות מתבצעת בצד השרת — לא ניתן לעקוף מצד הלקוח
  • CORS מוגבל לדומיין המערכת בלבד

4. אבטחת מידע — אחסון נתונים

  • כל הנתונים מאוחסנים ב-Google Firestore
  • הצפנה בדיסק (Encryption at Rest) — AES-256, מנוהל על ידי Google
  • אין גישה ישירה ל-Firestore מהדפדפן — רק דרך ה-Backend
  • Service Account עם הרשאות מינימליות (Principle of Least Privilege)
  • קבצי Service Account לא נשמרים ב-Git (כלולים ב-gitignore)
  • נתונים רגישים (סיסמאות, מפתחות) מנוהלים ב-Firebase/GCP — לא בקוד

5. אבטחת מידע — תקשורת

  • כל התקשורת מוצפנת ב-HTTPS/TLS — אין תעבורה ב-HTTP
  • Cloud Run מספק תעודת SSL אוטומטית (מנוהלת על ידי Google)
  • CORS מוגבל — רק הדומיין המורשה יכול לתקשר עם ה-API
  • Firebase Auth tokens מועברים בכותרת Authorization: Bearer
  • אין אחסון של מידע רגיש ב-cookies או localStorage
  • אין API חיצוני שמקבל מידע מזהה אישית (PII)

בקרות אבטחה ב-GCP

  • GCP Project ID: minlog-491211
  • Billing Account עם Budget Alerts מוגדרים
  • IAM — Service Account עם הרשאות מינימליות בלבד
  • אין גישה ציבורית ל-Firestore — רק דרך Service Account
  • Cloud Run — אימות ברמת האפליקציה (Firebase JWT)
  • Docker Images נבנים דרך Cloud Build (לא builds מקומיים בפרודקשן)

הסמכות GCP

  • SOC 1, SOC 2, SOC 3
  • ISO 27001 — ניהול אבטחת מידע
  • ISO 27017 — אבטחת מידע בענן
  • ISO 27018 — הגנה על מידע אישי בענן

8. בינה מלאכותית (AI)

המערכת כוללת צ'אטבוט לניתוח פיננסי מבוסס Google Gemini API.

מדיניות נתונים ב-AI

  • הצ'אטבוט מקבל רק סיכומי פרויקטים (שם, הכנסות, הוצאות, מרג'ין)
  • לא נשלחים נתוני שכר, פרטי ספקים או מידע אישי
  • Gemini API — הנתונים לא משמשים לאימון מודלים (לפי מדיניות Google)
  • בפיתוח: מעבר ל-Vertex AI שבו הנתונים נשארים בתוך ה-GCP Project
  • חישובים פיננסיים (What-If, Risk Score) מתבצעים בקוד Python — ללא AI
  • תשובות ה-AI הן ייעוציות בלבד — אין החלטות פיננסיות אוטומטיות

10. תאימות רגולטורית

Google Cloud Platform עומד בתקנים הבאים:

  • SOC 1, SOC 2, SOC 3 — בקרות ארגון שירות
  • ISO 27001 — מערכת ניהול אבטחת מידע
  • ISO 27017 — אבטחת מידע ספציפית לענן
  • ISO 27018 — הגנה על מידע אישי בענן ציבורי
  • GDPR — תאימות לרגולציה האירופית
  • Google Cloud DPA (הסכם עיבוד נתונים) — זמין לחתימה

  • 5. הגדרת Audit Log — מעקב אחרי פעולות רגישות

  • 7. בדיקת חדירות (Penetration Test) — אופציונלי
  • 8. הגדרת תהליך גיבוי ושחזור (RPO/RTO)
